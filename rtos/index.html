Real-Time OS

Schedule periodic real-time tasks using EDF, Rate Monotonic, and LLF. Verify schedulability and detect deadline misses.

Key Concepts

EDF
Earliest Deadline First — always runs the task with the closest upcoming deadline. Optimal for preemptive scheduling.
Rate Monotonic
Static priority: higher frequency = higher priority. Optimal among fixed-priority algorithms. Schedulable if U ≤ n(2^(1/n)−1).
LLF
Least Laxity First — runs the task with smallest slack time (laxity = deadline − remaining − current time).
Hyperperiod
LCM of all task periods. The full schedule repeats every hyperperiod.
Deadline Miss
A hard real-time task that misses its deadline causes system failure. Soft real-time tasks may degrade gracefully.
Schedulability
RM bound: U = Σ(Cᵢ/Tᵢ) ≤ n(2^(1/n)−1) → 0.693 for n→∞. EDF is schedulable if U ≤ 1.
