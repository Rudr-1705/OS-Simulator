Contiguous Memory Allocation

Allocate memory blocks using First Fit, Best Fit, Worst Fit, and Next Fit strategies. Visualize fragmentation in real time.

Key Concepts

First Fit
Allocates the first free hole that is large enough. Fast but can cause fragmentation at front.
Best Fit
Allocates the smallest hole that fits. Minimizes wasted space but leaves tiny unusable holes.
Worst Fit
Allocates the largest hole. Leaves larger remaining fragments but can waste big holes.
Next Fit
Like First Fit but starts from where it last left off. More uniform use of memory.
External Fragmentation
Total free memory is sufficient but not contiguous — no single hole fits the request.
Compaction
Shuffle memory contents to consolidate free space. Expensive — requires relocating all processes.
