File Allocation

Allocate disk blocks using Contiguous, Linked, and Indexed methods. See how each strategy places blocks across the disk.

Key Concepts

Contiguous
All blocks allocated sequentially. Fast random access (start + offset) but causes external fragmentation.
Linked
Each block has a pointer to the next. No fragmentation but sequential access only. FAT is an optimization.
Indexed
One index block holds all data block pointers. Supports random access, no fragmentation. inode in Unix.
Inode
Unix: index node holds file metadata + block pointers (direct, single/double/triple indirect).
FAT
File Allocation Table — linked allocation optimization. Linked list stored in a separate table, not in data blocks.
Fragmentation
Contiguous: external fragmentation over time. Linked/Indexed: internal fragmentation in last block only.
