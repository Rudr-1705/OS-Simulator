Key Concepts

Paging
Divides logical memory into fixed-size pages, physical into frames. Eliminates external fragmentation.
Page Table
Per-process map from virtual page numbers (VPN) to physical frame numbers (PFN).
Segmentation
Variable-size segments (code, data, stack). Each has base + limit. Supports logical protection.
Internal Fragmentation
Wasted space within a page/segment. Paging wastes up to (page_size - 1) bytes per process.
Segment Fault
Offset exceeds segment limit. Hardware raises protection fault → OS terminates or handles process.
Page Fault
Referenced page not in physical memory. OS must load it from disk — the most expensive memory operation.
