System Calls

A system call is how programs request help from the operating system. Since programs can't directly access files, memory, hardware, or create processes for security reasons, they use system calls. Here, execute different syscalls and watch how the kernel processes them.

Key Concepts

User Mode
Ring 3 — restricted access. Cannot directly access hardware or kernel memory.
Kernel Mode
Ring 0 — full access. OS code runs here. Entered via syscall instruction (trap).
Trap
Synchronous exception that transfers control from user to kernel. Sets privilege level to Ring 0.
Syscall Table
Kernel-maintained array of function pointers. Syscall number indexes into this table.
File Descriptor
Small non-negative integer representing an open file or I/O resource. Stdin=0, Stdout=1, Stderr=2.
Context Switch
Saving user registers, switching stack to kernel stack, loading kernel context on syscall entry.
