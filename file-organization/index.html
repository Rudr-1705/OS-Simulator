File System Simulation

Simulate directory structures and file sharing using DAG-based file system models.

Key Concepts

Single-Level
One directory for all files. Simple but prone to name conflicts. No hierarchy or file sharing.
Two-Level
Master directory with per-user sub-directories. Avoids name conflicts across users. Limited organisation.
Tree-Structured
Full hierarchy with nested subdirectories. Each file has exactly one parent. No native file sharing.
DAG (Directed Acyclic Graph)
Files may have multiple parent directories via hard links. Enables sharing without duplication. Requires reference counting for safe deletion.
Reference Count
Tracks how many directories point to a shared file. File is only deleted from disk when count reaches zero.
Hard Link
A directory entry pointing to the same inode as another entry. Both names refer to identical physical data on disk.

One flat directory; all files coexist. File names must be globally unique.
